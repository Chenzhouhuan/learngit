61 //
62 // Use implicit_cast as a safe version of static_cast or const_cast
63 // for upcasting in the type hierarchy (i.e. casting a pointer to Foo
64 // to a pointer to SuperclassOfFoo or casting a pointer to Foo to
65 // a const pointer to Foo).
66 // When you use implicit_cast, the compiler checks that the cast is safe.
67 // Such explicit implicit_casts are necessary in surprisingly many
68 // situations where C++ demands an exact type match instead of an
69 // argument type convertable to a target type.
70 //
71 // The From type can be inferred, so the preferred syntax for using
72 // implicit_cast is the same as for static_cast etc.:
73 //
74 // implicit_cast<ToType>(expr)
75 //
76 // implicit_cast would have been part of the C++ standard library,
77 // but the proposal was submitted too late. It will probably make
78 // its way into the language in the future.
79 template<typename To, typename From>
80 inline To implicit_cast(From const &f)
81 {
82  return f;
83 }

85 // When you upcast (that is, cast a pointer from type Foo to type
86 // SuperclassOfFoo), it's fine to use implicit_cast<>, since upcasts
87 // always succeed. When you downcast (that is, cast a pointer from
88 // type Foo to type SubclassOfFoo), static_cast<> isn't safe, because
89 // how do you know the pointer is really of type SubclassOfFoo? It
90 // could be a bare Foo, or of type DifferentSubclassOfFoo. Thus,
91 // when you downcast, you should use this macro. In debug mode, we
92 // use dynamic_cast<> to double-check the downcast is legal (we die
93 // if it's not). In normal mode, we do the efficient static_cast<>
94 // instead. Thus, it's important to test in debug mode to make sure
95 // the cast is legal!
96 // This is the only place in the code we should use dynamic_cast<>.
97 // In particular, you SHOULDN'T be using dynamic_cast<> in order to
98 // do RTTI (eg code like this:
99 // if (dynamic_cast<Subclass1>(foo)) HandleASubclass1Object(foo);
100 // if (dynamic_cast<Subclass2>(foo)) HandleASubclass2Object(foo);
101 // You should design the code some other way not to need this.
102

110  if (false)

112  implicit_cast<From*, To>(0);
